## 아쉬운 것 (Bad)

- **API 응답 포맷 오판**: 공공 데이터 포털 실제 응답 구조가 문서와 달라서 파싱 수정 커밋이 한 번 더 필요했다. 연동 전 실제 응답을 curl로 먼저 확인하는 습관이 필요하다.
- **목표 도메인 진행률 API 미완**: `GoalController`와 서비스는 만들었지만 진행률 계산 API(`/goals/{id}/progress`)는 다음 주로 넘어갔다. 목표 생성만 되고 진행 화면이 실데이터와 연결되지 않는 상태.

---

## 배운 것 (Learned)

- **한국 공공 데이터 포털 API**: 요청 시 `numOfRows`, `pageNo` 파라미터가 필수이고, 실제 필드명이 OpenAPI 문서와 다른 경우가 많다. 첫 연동 전에 `curl`로 응답을 직접 확인하는 것이 필수다.
- **Redis 캐시 설정**: `@Cacheable` + `CacheConfig` 조합 시 TTL을 `RedisCacheConfiguration`에서 명시적으로 설정해야 한다. 기본값이 무기한이라 운영에서 메모리 문제가 생길 수 있다.
