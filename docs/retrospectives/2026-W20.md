## 잘 된 것 (Good)

- **BLOCKER 순서대로 처리**: 의존 관계(DNS → HTTPS → ATS 예외 제거 → Bundle ID → App Icon)를 올바른 순서로 해결했다. 막힌 곳 없이 흘러갔다.
- **Terraform 한 방**: Stage 1 전체를 하나의 `apply`로 프로비저닝. 인프라 수동 설정 없이 코드로만 관리하는 구조가 잡혔다.
- **다크모드 일괄 처리**: 화면별로 패치하던 방식 대신 컬러 토큰 시스템으로 전환해 한 번에 전체 적용. 이후 추가 화면도 토큰만 쓰면 자동 지원된다.
- **xcconfig 트릭 발견**: `https:/$()/api.gainsy.site` 패턴으로 xcconfig에서 `//` 주석 처리 문제 우회. 타 iOS 프로젝트에서도 쓸 수 있는 패턴이다.

## 아쉬운 것 (Bad)

- **S3 prod 설정 삽질**: `LocalStack → host.docker.internal`, presigned URL 공개 엔드포인트, nginx 프록시 설정까지 S3 관련 수정이 4~5번 반복됐다. 로컬과 prod 환경 차이를 초기에 문서화했으면 줄일 수 있었던 비용이다.
- **dev → prod PR이 아직 미완**: 모든 BLOCKER를 dev 브랜치에서 처리했지만 prod merge가 빠졌다. 지금 prod 서버에서 502가 나는 이유다. 다음 턴 첫 번째 작업이어야 한다.
- **Xcode Archive는 수동만 가능**: Xcode Cloud 워크플로우 미설정 상태라 TestFlight 업로드를 자동화할 수 없다. CI로 빌드는 되지만 Archive → Distribute는 여전히 로컬 수동 작업이다.
- **동일 파일 여러 번 커밋**: 앱 이름 변경(`Vitae` → `gainsy`) 관련 커밋이 분산됐다. 한 번에 처리할 수 있었던 것을 2~3번 나눠서 작업했다.

## 배운 것 (Learned)

- **nginx `proxy_pass` 공백 버그**: nginx 설정에서 `proxy_pass http://localhost:8080 ;` (세미콜론 앞 공백)이 파싱 오류를 일으킨다. sed 패턴도 공백에 민감하다 — 설정 파일은 들여쓰기/공백을 엄격히 관리해야 한다.
- **`forward-headers-strategy: native`**: Nginx 뒤에 Spring Boot를 배치할 때 이 설정 한 줄로 `scheme`, `host`, `client-ip`를 올바르게 인식한다. 없으면 HTTPS 요청도 HTTP로 인식해 redirect 루프가 생긴다.
- **`PrivacyInfo.xcprivacy`는 App Store 필수**: 2024년부터 Apple이 필수로 요구하는 Privacy Manifest. 없으면 심사에서 거절된다. API 사용 이유(NSPrivacyAccessedAPITypes)를 반드시 선언해야 한다.
- **Route 53 호스팅 영역 비용**: 호스팅 영역 자체도 월 $0.50. 도메인을 여러 개 관리하면 비용이 쌓인다. 하나의 영역에 서브도메인으로 묶는 게 낫다.
