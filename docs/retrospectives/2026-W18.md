## 아쉬운 것 (Bad)

- **디바운스 관련 커밋 산재**: 식단 검색 디바운스 관련 수정이 여러 커밋에 분산됐다. `onChange` 즉시 호출 제거, 검색어 삭제 초기화 경로 통일, 느린 응답 역전 방지까지 한 번에 설계하고 커밋했으면 더 깔끔했다.
- **FCM 로컬 테스트 불가**: `@ConditionalOnProperty`로 로컬에서 스케줄러를 비활성화했지만, 실제 FCM 발송을 로컬에서 검증할 방법이 없다. Firebase Emulator 도입을 검토해야 한다.
- **AI 추정 결과 View 배지/disclaimer 표시 확인 필요**: 백엔드와 iOS 모델은 준비됐지만 실 디바이스에서 AI 추정 카드가 올바르게 렌더링되는지 회귀 검증이 남아 있다.

## 배운 것 (Learned)

- **NFC 정규화**: 한국어 텍스트를 DB에 저장할 때 NFC(Normalization Form C) 정규화를 적용해야 키보드 입력 방식(Mac vs Android vs iOS)에 따른 자모 결합 차이로 중복 등록이 생기는 문제를 예방할 수 있다. Java에서는 `Normalizer.normalize(text, Normalizer.Form.NFC)`.
- **`ObjectProvider<T>` 패턴**: Spring에서 선택적 의존성을 다룰 때 `@Autowired(required = false)` 대신 `ObjectProvider<T>`를 쓰면 `getIfAvailable()`로 안전하게 null 처리가 가능하고 테스트에서 mock 주입이 쉬워진다.
- **`uniqued(by:)` Swift**: Swift 표준 라이브러리에는 `uniqued(by:)`가 없다. 직접 구현하거나 `Array.Extension`으로 추가해야 한다. `Set`을 쓰면 순서가 보장되지 않으므로 검색 결과 정렬 유지에 주의해야 한다.
