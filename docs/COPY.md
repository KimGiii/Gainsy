# Health Care 카피 가이드

앱 내 모든 한국어 문구의 보이스·종결어미·CTA·이모지 정책을 정의한다.
새 화면을 만들거나 기존 카피를 수정할 때 이 문서를 기준으로 검토한다.

---

## 1. 보이스

**차분한 헬스 코치.** 친구도, 의사도, 시스템도 아니다.

- 사용자를 응원하지만 과장하지 않는다.
- 의학적 단정 표현을 피한다("효과 보장", "확실히 빠집니다" 금지).
- 데이터를 신뢰하게 만든다("이번 주 평균 580kcal 섭취" 같은 구체 수치).
- 친근감을 위해 반말이나 이모지로 도배하지 않는다.

### 예시

| ❌ 톤이 튐 | ✅ 표준 |
|---|---|
| "오늘도 화이팅이에요! 💪🔥" | "오늘도 한 걸음 나아갔어요" |
| "와! 단백질 목표 달성!! 🎉" | "단백질 목표를 채웠어요" |
| "운동 안 한 지 3일째다... 😢" | "최근 3일 동안 운동 기록이 없어요" |
| "AI가 분석한 결과..." | "이번 주 추세를 정리했어요" |

---

## 2. 종결어미 — **"~어요" 통일**

한 앱 안에 "없음 / 없어요 / 없습니다" 세 가지가 공존하지 않도록 한다.

| 상황 | 표준 어미 | 예시 |
|---|---|---|
| 빈 상태 | `~어요` | "아직 식단 기록이 없어요" |
| 안내 | `~어 주세요` | "키를 입력해 주세요" |
| 완료 알림 | `~었어요 / ~했어요` | "기록을 저장했어요" |
| 진행 중 | `~고 있어요` | "사진을 업로드하고 있어요" |
| 오류 | `~어요. 다시 시도해 주세요` | "불러오지 못했어요. 다시 시도해 주세요" |

### 예외 — 격식체 허용

의료·삭제·되돌릴 수 없는 동작은 격식체(`~합니다 / ~하시겠습니까`) 허용.

- "이 사진을 삭제하시겠습니까? 되돌릴 수 없습니다."
- "목표 유형은 변경할 수 없습니다."

### 금지

- ❌ `~없음` (체언 종결) — "목표 없음", "기록 없음"
- ❌ 반말 — "오늘도 잘했어!", "기록하자"
- ❌ 의문/감탄 남용 — "오늘 기록은요?!"
- ❌ "~할게요" (1인칭 의지) — 시스템이 아닌 사용자 목소리처럼 들림

## 3. CTA 라벨

### 주 CTA — **동사형 `~하기` 통일**

| ❌ 혼재 | ✅ 표준 |
|---|---|
| "저장" / "저장하기" | **"저장하기"** |
| "시작" / "시작하기" | **"시작하기"** |
| "다음" | **"다음으로"** |
| "추가" | **"추가하기"** |
| "기록" | **"기록하기"** |

### 보조 CTA — **2음절 명사 허용**

| 동작 | 표준 |
|---|---|
| 취소 | "취소" |
| 닫기 | "닫기" |
| 건너뛰기 | "건너뛰기" |
| 나중에 | "나중에" |

### 시스템 다이얼로그

- 긍정: "확인"
- 부정: "취소"
- 파괴적: **"삭제"** (빨강 강조) — `"삭제하기"` 아님, 다이얼로그에서는 명사형이 표준

## 4. 빈 상태 — 패턴 1종으로 통일

```
{대상} 기록이 아직 없어요
[+] {대상} 기록 시작하기
```

`EmptyState` 컴포넌트(예정)로 일관 적용한다.

| ❌ 현재 12종 혼재 | ✅ 표준 |
|---|---|
| "목표 없음" | "목표가 아직 없어요" |
| "설정된 목표가 없습니다" | (위와 동일) |
| "아직 기록 없음" | "오늘 식단 기록이 아직 없어요" |
| "측정 기록이 없습니다" | "신체 측정 기록이 아직 없어요" |
| "이 주에는 기록이 없어요" | (그대로 OK — 모범) |
| "진행률 데이터 없음" | "진행률 데이터가 아직 없어요" |

### 빈 상태 CTA

빈 상태에는 항상 다음 행동을 제시한다. CTA 없는 빈 화면 금지.

## 5. 알림 모달 (`.alert`)

### 제목 — **동사형 / 상태 명사형**

| ❌ | ✅ |
|---|---|
| `.alert("오류")` | `.alert("불러오지 못했어요")` 또는 `.alert("저장에 실패했어요")` |
| `.alert("사진 삭제")` | `.alert("사진을 삭제할까요?")` |

### 본문

- 최대 50자.
- 원인 + 사용자가 할 수 있는 다음 행동을 포함.
- "잠시 후 다시 시도해 주세요" 같은 모호 표현은 가능한 한 피한다.

### 푸시 알림

- 최대 60자.
- 시간 단서 + 동기부여 1줄.
- 예: "저녁 8시 · 오늘 운동 기록 잊지 마세요"
- 예: "이번 주 6일 연속 기록! 한 주 마무리해 볼까요?"

## 6. 이모지 정책

### 본문 텍스트에 이모지를 박지 않는다.

```swift
// ❌ 금지
Text("좋은 아침이에요 ☀️")
Text("개인 최고 기록 달성! 🎉")

// ✅ 허용 — 시각 요소를 별도 View로 분리
HStack {
  Image(systemName: "sun.max.fill")
    .foregroundStyle(Color.brandSunrise)
  Text("좋은 아침이에요")
}
```

### 예외 — 데이터 라벨

식사 카테고리(아침🍳/점심🥗/저녁🍽️) 같은 **데이터 속성**에서 이모지 허용. 단 emoji는 `MealType.emoji` 같은 모델 프로퍼티로 캡슐화하고 `Text` 인라인에 박지 않는다.

### 헤더 인사말 — 시간대 인사는 텍스트만

```swift
// ❌ 현재
"좋은 아침이에요 ☀️" / "활기찬 오후예요 💪" / "수고했어요 🌙"

// ✅ 변경
"좋은 아침이에요" / "활기찬 오후네요" / "오늘도 수고했어요"
// 시각 강조가 필요하면 옆에 별도 아이콘 컴포넌트 배치
```

## 7. 띄어쓰기 / 맞춤법

가장 자주 틀리는 항목.

| ❌ 자주 틀리는 형태 | ✅ 표준 |
|---|---|
| "시작해보세요" | **"시작해 보세요"** (보조용언 띄어쓰기) |
| "기록해보세요" | **"기록해 보세요"** |
| "삭제하시겠습니까" | OK (한 단어 어미) |
| "확인해주세요" | **"확인해 주세요"** |
| "안 됨" / "안됨" | 동사형: **"되지 않아요"** |
| "할 수 없습니다" | OK |

## 9. 도메인 용어 사전

같은 개념을 다른 단어로 부르지 않는다.

| ❌ 혼재 | ✅ 표준 |
|---|---|
| "운동/운동 기록/세션/트레이닝" | **"운동"** (기록 단위는 "운동 기록") |
| "식사/식단/끼니/밥" | **"식단"** (한 끼는 "식사") |
| "체중/몸무게" | **"체중"** (지표명), **"몸무게"** (입력 라벨) |
| "목표/목표치/타깃" | **"목표"** |
| "단백질/Protein/P" | **"단백질"** (본문), `P` (데이터 차트 축약 OK) |
| "여정/journey" | ❌ 사용 금지 — "기록"으로 대체 |
| "Pro/프리미엄/유료" | **"Pro"** |

## 10. 적용/검수 절차

### 새 화면 만들 때
1. 카피 초안 → 위 1·2·6·7장 자가 검토
2. PR에 카피 변경분 있으면 본 문서 링크 첨부

### 기존 화면 수정할 때
1. 같은 화면 내 종결어미·CTA가 본 문서와 충돌하면 같이 정리
2. 부분 수정만 하고 다른 곳의 일관성을 깨뜨리지 않는다

### 일괄 정리(향후)
- `EmptyState` 컴포넌트 신설 → 빈 상태 12종 일괄 치환
- CTA 라벨 grep 치환 PR (저장→저장하기 등)
- 이모지 본문 박힘 6건 일괄 제거
